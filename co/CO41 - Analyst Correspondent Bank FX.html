4.7 SQL Queries (Client-Side via wasSQLDSO)

-- Data retrieval for existing DEPO
SELECT base.cd, base.ccy, base.bknm, base.depo_type, base.bkcd,
       base.ctry_cd, to_char(base.depo_il,'YYYY/MM/DD') depo_il,
       NVL(ltrim(replace(to_char(bal.shw_clbs,'9,999,999,999,999,999.99'),' .00',' 0')), '0.00') shw_clbs,
       base.atten_no, base.depo_etc1, base.make_il, base.chng_il,
       base.mng_br, base.sts, base.af_cd, base.cbs_cd, base.depo_etc2,
       base.depo_etc3, base.dbs_cd, base.recon_yn
  FROM afex_rch_base base, afex_rch_bal bal
 WHERE base.cd = bal.cd(+) AND base.ccy = bal.ccy(+)
   AND base.make_il = bal.tr_il(+)
   AND base.cd = :depo_cd AND base.ccy = :ccy;

-- BIC validation
SELECT co_bkcd FROM acom_coh_base WHERE co_bkcd = :bkcd;

-- Duplicate check
SELECT cd, bkcd FROM afex_rch_base WHERE cd = :ccy AND bkcd = :bkcd;

-- GL code name lookup
SELECT enm FROM acom_com_actcd WHERE br_no = '0000' AND ac_cd = :ac_cd;

5. COO4100 — Entry/Modify/Terminate Service

5.1 Processing Flow

flowchart TD A[COO4100 START] --> B["CO41_Clear() Init FICO41, RCH_BASE, RCH_BAL, COH_BASE"] B --> C["CO41_Get() cmg_base → Extract all FML fields"] C --> D["CO41_1000_Inck() Validate BIC + DEPO code + currency"] D --> E{"proc_gb?"} E -->|"1" Insert| F["CO41E_2000_DBck() Validate BIC in acom_coh_base Auto-generate DEPO code"] E -->|"2" Update| G["CO41U_2000_DBck() Verify existing DEPO"] E -->|"9" Terminate| H["CO41T_2000_DBck() Verify existing DEPO for termination"] F --> I["CO41E_3000_Set() INSERT rch_base + rch_bal"] G --> J["CO41U_3000_Set() UPDATE rch_base"] H --> K["CO41T_3000_Set() SET sts='9' or terminate"] I & J & K --> L["CO41_Put() Return DEPO code"] L --> M[TPRETURN MSG_SUCCESS]

5.2 Input Validation (CO41_1000_Inck)

# Error Code Rule Message
1 1002 bkcd must be 11 chars "Depo BIC code is essential..."
2 1003 ccy must be 3 chars "Depo Currency code is essential..."

5.3 DEPO Code Auto-Generation (Insert Mode)

-- Generate next available DEPO code for the currency
SELECT NVL(MAX(cd), '0000')
  INTO :max_cd
  FROM afex_rch_base
 WHERE ccy = :ccy;
-- Increment to get next code (string + 1)

5.4 Insert Logic (CO41E_3000_Set)

1. Copy FICO41 → RCH_BASE struct
2. Set make_il = current date, sts = "0"
3. INSERT INTO afex_rch_base
   - If DUP → UPDATE
4. INSERT INTO afex_rch_bal (init balance = 0)
   - Set tr_il = make_il, shw_clbs = today_bs

5.5 Output Parameters

FML Field Description
FOCO41_CD Generated/existing DEPO code

6. COO4800 — Cancel Service

6.1 Processing Flow

flowchart TD A[COO4800 START] --> B["CO48_Clear() Init FICO48, RCH_BASE, RCH_BAL, RCH_PEND, PCMOVRD, SIS_TRAN"] B --> C["CO48_Get() cmg_base → cmg_getovrd → Extract cd, ccy, bkcd"] C --> D["CO48_1000_Inck() Validate BIC + DEPO code + currency"] D --> E["CO48_2000_DBck() Multi-step validation"] E --> F["CO48_3000_Set() UPDATE sts = '9' in afex_rch_base"] F --> G["CO48_4000_SIS() SIS Audit Log"] G --> H["CO48_Put()"] H --> I[TPRETURN MSG_SUCCESS]

6.2 Input Parameters

FML Field Description
FICO48_CD DEPO Code (4 chars)
FICO48_CCY Currency Code (3 chars)
FICO48_BKCD BIC Code (11 chars)

6.3 Multi-Step Cancellation Validation (CO48_2000_DBck)

flowchart TD A[Start Validation] --> B{"afex_rch_base exists?"} B -->|No| B1["Error 1005: Depo code not found"] B -->|Yes| C{"sts == '9'?"} C -->|Yes| C1["Error 1006: Already cancelled"] C -->|No| D{"afex_rch_bal exists?"} D -->|No| D1["Error 1007: Balance info not found"] D -->|Yes| E{"bal.sts == '9'?"} E -->|Yes| E1["Error 1008: Balance already cancelled"] E -->|No| F{"shw_clbs != 0?"} F -->|Yes| F1["Error 1009: Balance exists, cannot cancel"] F -->|No| G[Request Approval]

[!IMPORTANT]
A DEPO cannot be cancelled if it has a non-zero balance (shw_clbs != 0). This is a critical business rule to prevent data loss.

6.4 Pending Transaction Check (CO48_RchPend_Chk)

-- Currently commented out in main flow, but implemented:
SELECT count(*) INTO :count
  FROM afex_rch_pend
 WHERE cd  = :cd
   AND ccy = :ccy;
-- If count > 0 → Error 1014 "Succession transaction exists"

[!NOTE]
The CO48_RchPend_Chk() function is implemented but not called in the main flow (commented out). It checks for pending transactions before allowing cancellation.

6.5 Cancellation SQL

UPDATE afex_rch_base
   SET sts = '9'
 WHERE ccy = :ccy AND cd = :cd;
