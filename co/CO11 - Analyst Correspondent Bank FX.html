8. Backend Business Logic

8.1 COO1100 — Entry/Modify Service Flow

flowchart TD A[COO1100 START] --> B[CO11_Clear
Initialize all buffers] B --> C[CO11_Get
Extract FML fields to FICO11 struct] C --> D[CO11_1000_Inck
Input Validation] D --> E[CO11E_2000_DBck
Database Validation] E --> F[CO11E_3000_Set
INSERT or UPDATE] F --> G[CO11_4000_SIS
Write SIS audit log] G --> H[CO11E_Put
Return result to client] H --> I[TPRETURN MSG_SUCCESS] D -->|Fail| X[CO11_tprt_err
TPRETURN_ERROR] E -->|Fail| X F -->|Fail| X

Step 1: CO11_Clear() — Buffer Initialization

  • Zeroes out FICO11, COH_BASE, RCH_BASE structs
  • Resets SIS log variables

Step 2: CO11_Get() — FML → Struct Extraction

  • Calls cmg_base('O','N') for common initialization ('O'=online, 'N'=no daily batch)
  • Uses GET() macro to extract all 76+ FML fields into the FICO11 struct

Step 3: CO11_1000_Inck() — Input Validation Rules

# Error Code Rule Error Message / Key
1 1002 co_bkcd must be exactly 11 characters AERR60056
2 1003 co_gb must be '1', '2', or '3' AERR70005
3 1004 send_gb must be 'S', 'T', 'F', or 'M' AERR70074
4 1005 ctry_cd must be at least 2 characters AERR10242
5 1006 co_bknm1 is mandatory (min 1 char) AERR60090
6 1007 aba_no must be numeric (if provided) AERR91408
7 1008 If send_gb='T', telex_no1 is mandatory (inline message)
8 1009 bex_method must be valid code from F337 AERR70075
9 1010 bex_cd=1 requires bex_il ≥ 10 chars; otherwise bex_il must be empty AERR70076
10 1011 tkey_cd/tkey_il mismatch check AERR70077
11 1012 sign_cd/sign_il mismatch check AERR70078
12 1013 tc_yn/tc_il mismatch check AERR70079
13 1014 ar_cd/ar_il mismatch check AERR70080
14 1015 If depo_chk='1', depo_cd is mandatory AERR70081

Step 4: CO11E_2000_DBck() — Database Validation

-- Check if BIC already exists
SELECT * INTO :COH_BASE FROM acom_coh_base WHERE co_bkcd = :co_bkcd;

-- If found and cond_sts = '9' → Error "Already canceled BIC Code"
-- If Depo=No and depo_cd is not empty:
SELECT COUNT(*) FROM afex_rch_base WHERE cd = :depo_cd AND sts = 0;
-- If count > 0 → Error "CO41 settlement offset needed first"

-- If Depo=Yes and depo_cd is not empty:
SELECT COUNT(*) FROM afex_rch_base WHERE cd = :depo_cd AND bkcd = :co_bkcd AND sts = 0;
-- If count = 0 → Error "Depo Code Not Found"

Step 5: CO11E_3000_Set() — INSERT/UPDATE Logic

  1. Copy all FICO11 fields → COH_BASE struct (with cmg_qstr() quotation escape for names/addresses)
  2. Set defaults: cond_sts = "0" (new), meet_no = 0, co_cd = ""
  3. Set operator/manager from global: op_no = G_FXFIG.opno, mng_no = G_FXFIG.mnno
  4. Attempt INSERT INTO acom_coh_base VALUES(:COH_BASE)
  5. If DUP (duplicate key):
    • Change cond_sts = "1" (modified)
    • Execute UPDATE acom_coh_base SET ... WHERE co_bkcd = :co_bkcd

[!NOTE]
This implements an UPSERT pattern: INSERT first, and if the PK already exists (DUP), fall back to UPDATE.

Step 6: CO11_4000_SIS() — SIS Audit Log

Sets SIS_TRAN structure with:

  • ysloggb = "1" (normal log)
  • yscnlgb = "" (not cancel)
  • ysinpgb = "5" (transaction type)
  • yscntgb = "1" (workload target)
  • ysbrno = G_FXFIG.ajum (branch number)
  • Calls cmg_sis_isrt() to write audit trail

8.2 COO1800 — Cancel Service Flow

flowchart TD A[COO1800 START] --> B[CO18_Clear
Initialize buffers + PCMOVRD] B --> C[CO18_Get
Extract BIC code only] C --> D[CO18_1000_Inck
BIC = 11 chars] D --> E[CO18_2000_DBck
Check existence + approval request] E --> F{Approval
Required?} F -->|Yes| G[TPRETURN_ERROR
'Approval Requesting'] F -->|No| H[CO18_3000_Set
UPDATE cond_sts='9'] H --> I[CO18_4000_SIS
Write cancel SIS log] I --> J[CO18_Put
Return result] J --> K[TPRETURN MSG_SUCCESS]

Key Differences from COO1100:

Aspect COO1100 (Entry) COO1800 (Cancel)
Input parameters 76 FML fields 1 field (co_bkcd only)
DB operation INSERT / UPDATE UPDATE cond_sts = '9'
Approval Not required Required (책임자승인 via cmg_ovrditemadd)
SIS cancel flag yscnlgb = "" yscnlgb = "1"
Status output cond_sts = "0" or "1" cond_sts = "9"
Input struct FICO11_S (93 fields) FICO18_S (1 field: co_bkcd)

Approval Logic (책임자승인):

// CO18_2000_DBck() → requests supervisor approval
cmg_ovrditemadd(&PCMOVRD, "11", "Corres Bank Cancel...", "");

// Main flow checks approval status
if (PCMOVRD.ovrd_gb[0] == NULL && PCMOVRD.ovrd_cc > 0 && G_FGINP.tr_gb[0] != '1') {
    PCMOVRD.ovrd_gb[0] = '1';
    cmg_putovrd(&PCMOVRD);
    TPRETURN_ERROR("Approval Requesting", 0);  // Suspend until approved
}

9. Database Trigger (coh_base_trg)

CREATE OR REPLACE TRIGGER coh_base_trg
    AFTER INSERT OR UPDATE ON coh_base FOR EACH ROW
DECLARE
    x_base_dt DATE;
BEGIN
    SELECT today INTO x_base_dt FROM com_dayinf;
    
    -- Try UPDATE ts_coh_base first
    UPDATE ts_coh_base SET ... WHERE base_dt = x_base_dt AND co_bkcd = :old.co_bkcd;
    
    -- If no row updated → INSERT
    IF SQL%ROWCOUNT = 0 THEN
        INSERT INTO ts_coh_base VALUES (x_base_dt, :new.co_bkcd, ...);
    END IF;
END;

[!TIP]
The trigger implements daily snapshots. Each business day (com_dayinf.today) gets at most one history row per BIC code in ts_coh_base.

14. Key Observations for New FX Project Development

[!WARNING]
The following points are critical considerations when re-developing this module for the new Foreign Exchange project.

14.1 Technology Migration

  • Frontend: Migrate from ActiveX/IE to modern web framework (React/Vue/Angular)
  • Middleware: Replace Tuxedo/FML with REST API or gRPC
  • Backend: Replace Pro*C with Java Spring Boot (JPA/MyBatis)
  • Database: Oracle tables remain but need DDL scripts for acom_coh_base, ts_coh_base, afex_rch_base

14.2 Business Logic to Preserve

  1. UPSERT pattern: INSERT first → UPDATE on DUP
  2. BIC validation: Exactly 11 characters, auto-append 'XXX' for 8-char input
  3. Cross-field validation: All 5 exchange checkbox/date pairs
  4. DEPO cross-validation against afex_rch_base
  5. Supervisor approval for cancel operations
  6. SIS audit log for all transactions
  7. Trigger-based history (daily snapshot to ts_coh_base)

14.3 Code Table Dependencies

  • F337: Exchange Method values (loaded via cmg_codeck)
  • acom_com_ctry: Country code/name lookup
  • com_dayinf: Current business date

14.4 Missing/Unused Features

  • co_cd (Center Corres Code) is always set to "" on registration
  • FICO11_MEET_NO is not sent from frontend (set to 0 server-side)
  • SIS data copy to G_sisbuf is disabled (#if 0)
  • telex_no5 exists in FML but not explicitly in UPDATE columns (only in INSERT via struct)
