1. Penjelasan untuk Orang Awam

🤔 Apa Sih Modul CO41 Ini?

Bayangkan Anda punya Bank OK di Jakarta. Bank OK ingin bisa mengirim dan menerima uang dalam mata uang asing (USD, EUR, JPY, dll). Masalahnya, Bank OK tidak punya kantor di New York atau London. Jadi bagaimana caranya?

Jawabannya: Bank OK membuka "rekening titipan" di bank luar negeri.

Modul CO41 adalah sistem pencatatan rekening titipan tersebut — siapa bank-nya, di mana, mata uang apa, nomor rekeningnya berapa, dan berapa saldonya.

🏦 Analogi Sederhana

┌─────────────────────────────────────────────────────────────┐
│                    DUNIA NYATA                              │
│                                                             │
│  Anda (individu) membuka rekening di Bank BCA              │
│  → Anda punya "buku tabungan" yang mencatat saldo Anda     │
│                                                             │
│  Bank OK (bank) membuka rekening di Citibank New York       │
│  → Bank OK punya "DEPO ledger" yang mencatat saldo mereka  │
│                                                             │
│  Modul CO41 = "Sistem pendaftaran buku tabungan" Bank OK    │
│               di bank-bank luar negeri                      │
└─────────────────────────────────────────────────────────────┘

💱 Apa itu Nostro dan Vostro?

Ini adalah istilah perbankan internasional yang terdengar rumit, tapi sebenarnya sederhana:

Istilah Bahasa Latin Artinya Contoh

Vostro "Milik Anda" Rekening bank LAIN di KITA Citibank punya rekening IDR di Bank OK

🌍 Contoh Transfer Dunia Nyata

Skenario: Pak Budi di Jakarta ingin kirim $10,000 ke Pak John di New York

sequenceDiagram participant Budi as Pak Budi
(Nasabah Bank OK) participant OK as Bank OK
(Jakarta) participant DEPO as Nostro Account
(Rek. Bank OK di Citibank) participant Citi as Citibank
(New York) participant John as Pak John
(Nasabah Citibank) Budi->>OK: "Saya mau kirim $10,000 ke New York" OK->>OK: Potong rekening Pak Budi (IDR) OK->>DEPO: Instruksi: "Kurangi saldo Nostro $10,000" DEPO->>Citi: SWIFT MT103 (Perintah transfer) Citi->>John: Kredit rekening Pak John $10,000 Note over DEPO: Saldo Nostro berkurang $10,000

Tanpa Nostro Account, transfer ini TIDAK BISA terjadi!

[!IMPORTANT]
Inilah mengapa modul CO41 sangat penting — ia mendaftarkan dan mengelola "rekening titipan" (Nostro/Vostro) yang menjadi jembatan untuk semua transaksi valuta asing internasional.

4.2 Validasi Input Detail (CO41_1000_Inck)

16 aturan bisnis yang dicek sebelum data disimpan:

# Kode Apa yang Dicek Analogi Dunia Nyata
1 1003 Mata uang harus 3 huruf "USD", "EUR" — bukan "US" atau "Dollar"
2 1004 Mata uang harus valid (ada di code table) Tidak boleh asal ketik "XYZ"
3 1005 Tipe proses harus 1, 2, atau 9 Hanya bisa: Insert, Update, atau Terminate
4 1006 Tipe Depo harus 1, 2, atau 3 Hanya bisa: Vostro, Nostro Domistik, Nostro Overseas
5 1007 BIC Code harus 11 karakter Alamat SWIFT bank koresponden wajib
6 1008 BIC Code harus valid (ada di system) Bank yang dituju harus terdaftar
7 1009 Nomor rekening wajib diisi Seperti buka tabungan — harus ada nomor rekening
8 1010 Kode aset dan liabilitas wajib Untuk pencatatan pembukuan bank
9 1011 Vostro: Asset Code harus mulai '2' Aturan akuntansi untuk rekening Vostro
10 1012 Vostro: Liability Code harus mulai '1' Aturan akuntansi untuk rekening Vostro
11 1013 Nostro: Asset Code harus mulai '1' Aturan akuntansi untuk rekening Nostro
12 1014 Nostro: Liability Code harus mulai '2' Aturan akuntansi untuk rekening Nostro
13 1015 DEPO code after termination harus cocok prefix Kalau tutup rekening, penggantinya harus tipe yang sama
14 1016 Kode cabang wajib Harus jelas cabang mana yang mengelola
15 — Reconcile default = "N" jika kosong Auto-set untuk keamanan

[!TIP]
Aturan Akuntansi Nostro vs Vostro:

  • Nostro (uang kita di bank lain) = Aset bagi kita → kode aset harus dimulai '1' (aktiva)
  • Vostro (uang bank lain di kita) = Kewajiban bagi kita → kode liability harus dimulai '1' (hutang)

4.4 Operasi Database (CO41E_3000_Set)

Insert (proc_gb = 1)

-- 1. Cek duplikat (BIC + CCY + Account Number)
SELECT CD FROM afex_rch_base 
 WHERE ccy = :ccy AND bkcd = :bkcd AND atten_no = :atten_no;
-- Jika ada → Error 1030 "Already Inserted"

-- 2. Generate DEPO code (lihat logika di atas)

-- 3. INSERT rekening baru
INSERT INTO afex_rch_base VALUES (:RCH_BASE);

-- 4. INSERT saldo awal
INSERT INTO afex_rch_bal VALUES (:RCH_BAL);
-- saldo awal = today_bs, confirm = 'N'

-- 5. Update corres master → link DEPO code
UPDATE acom_coh_base 
   SET depo_chk = '1', depo_cd = :depo_cd
 WHERE co_bkcd = :bkcd;

Update (proc_gb = 2) atau Terminate (proc_gb = 9)

-- UPDATE semua kolom (16 field) berdasarkan PK
UPDATE afex_rch_base
   SET bkcd = :bkcd, bknm = :bknm, ctry_cd = :ctry_cd,
       depo_il = :depo_il, vostro_yn = :yn, atten_no = :acct,
       sts = :sts,  -- '0' untuk update, '9' untuk terminate
       ...
 WHERE ccy = :ccy AND cd = :cd;

[!WARNING]
Terminate vs Cancel: Ada perbedaan penting!

  • Terminate (proc_gb=9 di COO4100): Operator memilih "Terminate" → set sts='9' — memerlukan saldo = 0
  • Cancel (COO4800): Fungsi terpisah → set sts='9' — juga memerlukan saldo = 0

5.2 Alur Cancel

flowchart TD A["COO4800 dipanggil"] --> B["CO48_Clear()"] B --> C["CO48_Get()
Baca CD, CCY, BKCD"] C --> D["CO48_1000_Inck()
Validasi 3 field"] D --> E["CO48_2000_DBck()

5.3 Lima Pengecekan Sebelum Cancel

-- 1. Cek rekening ada
SELECT * FROM afex_rch_base WHERE ccy = :ccy AND cd = :cd;
-- Jika bkcd kosong → Error 1005 "not found"

-- 2. Cek belum dibatalkan
-- Jika sts = '9' → Error 1006 "already canceled"

-- 3. Cek balance record ada
SELECT * FROM afex_rch_bal WHERE cd = :cd AND ccy = :ccy AND tr_il = :today;
-- Jika kosong → Error 1007 "balance not found"

-- 4. Cek balance belum cancelled
-- Jika bal.sts = '9' → Error 1008 "balance already canceled"

-- 5. Cek saldo = 0
-- Jika shw_clbs != 0 → Error 1009 "Balance exist — cannot cancel"

5.4 SIS Audit Log (Jejak Audit)

Cancel service menulis log audit lengkap ke sistem SIS (Security Information System):

Field SIS Nilai Arti
ysloggb "1" Normal log
yscnlgb "1" Ini adalah transaksi pembatalan
ysinpgb "5" Tipe transaksi = 5
ysbrno G_FXFIG.ajum Cabang yang melakukan
yscntgb "1" Dihitung sebagai workload

[!NOTE]
Mengapa Cancel punya SIS tapi Entry tidak? Karena pembatalan rekening adalah operasi berisiko tinggi — harus ada jejak audit siapa yang membatalkan, kapan, dan dari cabang mana.

8. Skenario Dunia Nyata

📖 Skenario 1: Buka Rekening USD di Citibank New York

Konteks: Bank OK ingin bisa mengirim/menerima USD via Citibank.

Langkah di Sistem CO41:

  1. Operator pilih Processing Type = Insert
  2. Isi data:
    • Ccy = USD
    • Depo Type = Nostro (Overseas) — karena rekening Bank OK di bank luar negeri
    • BIC Code = CITIUS33XXX
    • Depo Name = CITIBANK N.A.
    • Country = US
    • A/C No = 36829754
    • Asset Code = 110201 (harus mulai '1' karena Nostro = aset)
    • Liability Code = 210301 (harus mulai '2')
    • Branch = 210
    • Reconcile = Y (Citibank kirim MT950 otomatis)
  3. Klik Submit
  4. Sistem otomatis generate DEPO code: 3001
  5. Saldo awal: $0.00
  6. Bank koresponden acom_coh_base di-update: depo_chk = 1

📖 Skenario 2: Update Informasi Rekening

Konteks: Citibank pindah cabang, nomor rekening berubah.

  1. Operator pilih Processing Type = Update
  2. Masukkan DEPO Code 3001, CCY USD
  3. Sistem otomatis load data lama
  4. Ubah A/C No dari 36829754 ke 48201573
  5. Klik Submit → data di-UPDATE

📖 Skenario 3: Tutup Rekening (Terminate)

Konteks: Bank OK tidak lagi menggunakan Citibank, pindah ke JP Morgan.

  1. Prasyarat: Saldo harus nol! ($0.00)
    • Jika saldo masih ada → Error: "Balance is not zero"
  2. Operator pilih Processing Type = Terminate
  3. DEPO Code = 3001, CCY = USD
  4. Isi "After Termination" = 3002 (DEPO baru di JP Morgan)
  5. Submit → status berubah ke sts = '9' (terminated)

📖 Skenario 4: Cancel (Pembatalan Registrasi)

Konteks: Operator salah daftarkan rekening, perlu dibatalkan.

  1. Operator masuk mode Cancel (OPStatus = 1)
  2. Masukkan DEPO Code, CCY, BIC
  3. Sistem cek:
    • ✅ Rekening ada
    • ✅ Belum di-cancel sebelumnya
    • ✅ Saldo = 0
  4. Submit → Cancel → SIS audit log ditulis
  5. Status sts = '9'
